# Roadmap

MVP 以降の UI 改善・機能追加方針。優先度は上から高い順。着手前にこのファイルを更新し、完了したら取り消し線で履歴として残す運用を想定。

---

## 短期（1〜2週間）— MVP の使い勝手を底上げ

1. **AI 分析の前面化**
   - 銘柄詳細で「直近の AI 判断」を最上段に出し、スコア・推奨度・通期見通し・確信度を一目で見える形に。
   - 過去の分析はその下にサブカードで畳む。
2. **通知の説明強化**
   - 通知本文に「なぜ送られたか」（閾値 / 適時開示 / スコア）を明記。
   - クリックで該当分析（`/ticker/{ticker}` または `/history` の該当行）へ遷移。
3. **チャートの情報量追加**
   - 出来高バー、5 / 25 / 75 日移動平均線。
4. **ダッシュボード「今日の注目」**
   - 自分のウォッチから「最も大きく下げた」＋「最近 AI 分析でスコアが跳ねた」の 2 ピックを固定枠に表示。
5. **祝日対応**
   - JP 祝日判定で cron の空振りをスキップ（例: `@holiday-jp/holiday_jp`）。

## 中期（1〜2ヶ月）— 意思決定支援の強化

1. **ポートフォリオ機能**
   - 保有株・取得単価を登録 → 含み損益・配当利回り表示。ウォッチから保有に昇格できるフロー。
2. **スクリーニング**
   - PER / PBR / 配当利回り / 時価総額でウォッチ候補をフィルタ。
3. **過去分析との差分ハイライト**
   - 前回からスコア・構造分類が変わった銘柄をダッシュボードで強調。
4. **週次サマリメール**
   - 金曜夕方に「今週のハイライト（AI 分析・急落・通知数）」を配信。
5. **複数銘柄比較チャート**
   - 任意の 2〜5 銘柄をオーバーレイ描画。

## 長期（3ヶ月〜）— 情報の質を一段上へ

1. **本文取得強化**
   - 有価証券報告書・決算説明会資料の本文を取り込み、AI 分析の確信度を底上げ（現状 `docDescription` のみで `confidence: low` が多い）。
2. **セクター・市場全体の俯瞰**
   - 日経主要銘柄のスコア分布・下落率ヒートマップ。
3. **チャット履歴と横断質問**
   - ウォッチ全銘柄に対して「今週の決算で割安に見える3銘柄は？」のような横断質問。
4. **PWA 強化**
   - オフライン表示、ホーム追加ガイド、iOS Web Push 対応。
5. **観測性**
   - Sentry / Vercel Observability で cron 失敗・API エラーを可視化。
